# 3.7. Boltz-1

[[Home|Home]] > [[EN/3. Models/3.0. Models Overview|Models]] > Boltz-1
🇺🇦 [[UA/3. Моделі/3.7. Boltz-1|Українська]]

`Boltz-1` is an open biomolecular interaction model positioned as an `AF3`-adjacent system for predicting 3D biomolecular complexes. Its main importance is that it moves multimolecular complex prediction from a server-first paradigm toward an open local stack.

## Why Boltz-1 matters

Many researchers want `AF3`-like capabilities but also need:

- open weights;
- trainable code;
- local inference;
- inspectable data and benchmark pipelines.

Boltz-1 matters because it tries to provide all of these together in one open package.

## Practical role in a local stack

Boltz-1 is best understood as an open generalist for biomolecular interactions:

- broader than a pure protein-only model;
- more local and modifiable than a server-only workflow;
- a natural comparison point next to `AF3` and `Chai-1`.

## Strengths

| Strength | Why it matters |
| --- | --- |
| Open code and weights | Easier to inspect, benchmark, and automate locally |
| Biomolecular-complex focus | Relevant for proteins, nucleic acids, and ligands in one workflow |
| MIT license | Strong practical value for research software reuse |
| Fast-moving ecosystem | Useful for open benchmarking and engineering iteration |

## When Boltz-1 is useful

- when an open `AF3`-adjacent baseline is needed;
- when local inference and automation matter more than staying inside a hosted server workflow;
- when the team wants a modifiable biomolecular-complex stack;
- when direct comparison with `Chai-1` or `AF3` is part of the evaluation plan.

## Limitations

| Limitation | Why it matters |
| --- | --- |
| Preprint-stage literature | Claims should be treated more cautiously than long-established peer-reviewed baselines |
| Rapidly evolving software | Practical behavior can change faster than classic benchmark papers |
| More complex than protein-only stacks | Multimolecular flexibility increases operational complexity |
