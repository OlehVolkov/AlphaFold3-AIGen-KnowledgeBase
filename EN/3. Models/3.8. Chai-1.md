# 3.8. Chai-1

[[Home|Home]] > [[EN/3. Models/3.0. Models Overview|Models]] > Chai-1
🇺🇦 [[UA/3. Моделі/3.8. Chai-1|Українська]]

`Chai-1` is a multi-modal foundation model for molecular structure prediction. In practice it is one of the most important open local alternatives for predicting complexes that mix proteins, small molecules, DNA, RNA, glycosylations, and user-provided restraints.

## Why Chai-1 matters

`AF3` made broad biomolecular interaction modeling mainstream, but many local workflows still need:

- open code and weights;
- programmable inference;
- explicit control over templates, MSAs, and restraints;
- a path to local deployment on GPU hardware.

Chai-1 is important because it packages these needs into a practical open stack.

## Main capabilities

| Capability | Practical meaning |
|---|---|
| Multi-modal input | Proteins, ligands, DNA, RNA, glycosylations, and more |
| Local inference | Useful for scripted batch workflows and reproducible pipelines |
| MSA support | Can run with default embeddings, but can also use richer MSA input |
| Restraints and custom contexts | Helpful for guided modeling and integrative setups |

## When Chai-1 is useful

- when a local complex-prediction stack is preferred over a hosted service;
- when modeling requires additional restraints or custom template control;
- when one workflow needs proteins together with ligands and nucleic acids;
- when an open benchmark against `AF3` or `Boltz-1` is needed.

## Strengths

- **Broad entity coverage** in one workflow;
- **Practical local interface** with CLI and Python entry points;
- **Open weights and code** under a permissive license;
- **Better fit for automation** than a server-only workflow.

## Limitations

| Limitation | Why it matters |
|---|---|
| Hardware-heavy | Large complexes still require strong GPU resources |
| Fast-changing implementation | Practical details can change quickly in an active repo |
| Still a young benchmark target | Real-world behavior should be validated per task class |
