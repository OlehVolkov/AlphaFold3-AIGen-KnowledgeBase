# Diffusion Models

[[Home|Home]] > [[EN/Index|Concepts]] > Machine Learning
🇺🇦 [[UA/2. Концепції/2.2. Машинне-Навчання/2.2.2. Дифузійні моделі|Українська]]

Diffusion models generate samples by reversing gradual noise injection.

$$q(x_t|x_0)=\mathcal{N}(\sqrt{\bar\alpha_t}x_0,(1-\bar\alpha_t)I)$$

## Brief overview

- Forward process: data to Gaussian noise
- Reverse process: denoising model learns to reconstruct samples
- Training target: noise prediction or score prediction

## U-Net in image diffusion

`U-Net` is an encoder-decoder architecture with skip connections between symmetric resolution levels:

- **Encoder path**: downsampling with increasing channel depth.
- **Bottleneck**: compact global representation.
- **Decoder path**: upsampling back to target resolution.
- **Skip connections**: preserve fine spatial details and improve reconstruction.

In image diffusion (DDPM/latent diffusion), U-Net typically parameterizes $\varepsilon_\theta(x_t, t, c)$ or $v_\theta(\cdot)$.

## Comparison: AF3 diffusion vs image diffusion (U-Net)

| Aspect | AF3 diffusion | Image diffusion (U-Net) |
|---|---|---|
| State space | 3D atomic coordinates $\mathbb{R}^{N\times3}$ | Pixels or latent maps $\mathbb{R}^{H\times W\times C}$ |
| What is denoised | Molecular complex geometry | Image/latent tensor |
| Core denoiser | AF3 geometry-aware module with Pairformer conditioning | U-Net (encoder-decoder + skip connections) |
| Conditioning | Single/pair representations, molecule types, structural context | Text, class labels, image conditioning (often via cross-attention) |
| Symmetry constraints | Strong $SE(3)$ consistency requirements in 3D | Primarily 2D grid inductive bias; full rotation equivariance is usually not explicit |
| Training target | Noise/score in coordinate space + structural constraints | Noise $\varepsilon$, velocity $v$, or score in 2D/latent space |
| Output | 3D biomolecular complex structure | Synthesized image |
| Typical metrics | RMSD, lDDT/pLDDT, DockQ, clash/contact quality | FID, CLIP score, human preference |

In short, both use reverse diffusion, but the **inductive biases** differ: AF3 prioritizes physically consistent 3D geometry, while image diffusion prioritizes multi-scale 2D structure and semantic conditioning.

## LoRA for diffusion backbones

`LoRA` can be applied to diffusion models with the same low-rank update idea used in transformers:

$$W' = W + \Delta W, \qquad \Delta W = \frac{\alpha}{r}BA$$

### Why it matters in diffusion

- Full fine-tuning of `U-Net`, `DiT`, or text-conditioning blocks is expensive.
- Many practical goals are narrow: style transfer, subject personalization, domain adaptation, or control-specific tuning.
- A small LoRA checkpoint is easier to store and mix than a full diffusion checkpoint.

### Common insertion patterns

| Diffusion backbone | Common LoRA targets | Typical use |
|---|---|---|
| Latent diffusion `U-Net` | cross-attention `q/k/v/o` projections, sometimes feed-forward or lightweight convolutional blocks | style/subject adaptation |
| Diffusion Transformer (`DiT`) | self-attention and MLP projections | domain transfer with transformer-like tooling |
| Multimodal conditioning stack | text encoder attention/MLP layers | adapting prompt understanding or alignment |

### Main properties

- LoRA changes only selected layers, so the base denoiser stays frozen.
- Adapter files are small enough to distribute independently from the base model.
- For image diffusion, LoRA often targets conditioning-sensitive layers first because they control prompt-to-image behavior strongly.
- For diffusion transformers, the workflow becomes very similar to LoRA on LLMs.

### Limitations

- Over-specialized adapters can reduce diversity or oversaturate a style.
- Concept leakage is common when the training set is small or poorly curated.
- Mixing many adapters can create interference between prompt semantics and denoising behavior.

## Quantization for diffusion models

Quantization is harder for diffusion than for one-pass classifiers because denoising errors can accumulate over many timesteps.

### Why diffusion quantization is harder

- Activation distributions change with timestep $t$.
- The denoiser is called repeatedly, so a small error at one step can amplify later.
- Skip or shortcut paths may show distinct activation statistics from the main branch.

### Main approaches

| Approach | Typical target | Why it is used | Main risk |
|---|---|---|---|
| PTQ with timestep-aware calibration | pretrained `U-Net` / `DiT` | no expensive retraining | calibration mismatch across timesteps |
| Mixed precision | sensitive blocks stay at higher precision | protects image quality | weaker compression |
| Weight-only low-bit inference | memory-heavy denoiser weights | cheaper deployment | activations still expensive |
| Quantization + fewer-step samplers/distillation | full sampling loop | latency reduction on top of compression | harder quality tuning |

### Transformers vs diffusion: what changes

| Aspect | Transformer quantization | Diffusion quantization |
|---|---|---|
| Forward passes per sample | usually one decoding step at a time | tens of denoising steps |
| Error accumulation | local to next tokens | compounds over the sampling trajectory |
| Calibration target | token activations / sequence statistics | timestep-dependent latent statistics |
| Sensitive modules | attention, MLP, KV cache | denoiser blocks, timestep embeddings, skip paths |

### Practical rule of thumb

- For transformer-based diffusion backbones (`DiT`), reuse many transformer quantization ideas.
- For `U-Net` diffusion, calibrate with representative timesteps instead of only static activations.
- Start with weight quantization before aggressive activation quantization.
- If quality drops sharply, move the most sensitive blocks back to higher precision.
