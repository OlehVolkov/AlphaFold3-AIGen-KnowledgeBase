# 1.5.7. Foldseek and Structure Search

[[Home|Home]] > [[EN/1. AlphaFold3/1.5. Resources/1.5.1. Key Terms|Resources]] > Foldseek and Structure Search
🇺🇦 [[UA/1. AlphaFold3/1.5. Ресурси/1.5.7. Foldseek і пошук структур|Українська]]

`Foldseek` is a practical structure-search system for comparing large sets of experimental and predicted structures. In an `AF2`/`AF3` era, this matters because structure prediction is only half of the workflow; the other half is finding similar folds, assemblies, and plausible analogs at scale.

## Why Foldseek matters

Once a predicted structure exists, a common next question is:

- has a similar fold already been seen;
- does a related interface exist in `PDB` or `AlphaFoldDB`;
- can a predicted complex be compared quickly against a large structure collection.

Foldseek matters because it makes this kind of search practical at database scale.

## Core use cases

| Use case | Why it matters |
|---|---|
| Fast structure search | Find related folds more efficiently than slow pairwise structural alignment workflows |
| Multimer search | Search at assembly level rather than only single chains |
| Clustering | Organize large predicted-structure collections |
| Predicted-structure workflows | Useful with `PDB`, `AlphaFoldDB`, and local predicted models |

## Practical value for this vault

Foldseek is especially relevant here because the knowledge base already mixes:

- experimental structures from `PDB`;
- computed models from `AlphaFoldDB`;
- local outputs and PDF-derived references.

That makes a structure-search note more useful than a purely theoretical comparison.

## Limitations

| Limitation | Why it matters |
|---|---|
| Search is not interpretation | A structural hit still needs biological validation |
| Fast heuristics are not full mechanistic proof | Similar folds do not always imply shared function |
| Best for retrieval, not generation | Foldseek complements prediction models; it does not replace them |
