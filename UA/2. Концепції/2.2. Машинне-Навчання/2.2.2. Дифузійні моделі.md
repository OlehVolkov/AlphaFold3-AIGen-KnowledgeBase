## Короткий огляд

**Дифузійні моделі** (DDPM) генерують дані через зворотній процес видалення шуму:

$$\boxed{q(x_t|x_0) = \mathcal{N}(\sqrt{\bar\alpha_t}\,x_0,\,(1-\bar\alpha_t)\mathbf{I})} \quad \longleftrightarrow \quad p_\theta(x_{t-1}|x_t) = \mathcal{N}(\mu_\theta,\Sigma_\theta)$$

## В контексті AlphaFold 3

AF3 генерує **атомні координати** $\mathbf{X}\in\mathbb{R}^{N_\text{atoms}\times3}$ через дифузію з conditioning від Pairformer (single $s$ + pair $z$). Це дозволяє моделювати весь молекулярний комплекс — білки, ДНК, РНК, ліганди — в одному процесі.

## U-Net для генерації зображень

`U-Net` (U-подібна мережа) — енкодер-декодер з `skip connections` між симетричними рівнями:

- **Encoder path**: зменшує просторову роздільність і збільшує число каналів.
- **Bottleneck**: містить глобальніші ознаки.
- **Decoder path**: відновлює роздільність.
- **Skip connections**: повертають деталі з ранніх шарів для точного відновлення структури.

У diffusion-моделях для зображень (DDPM/Latent Diffusion) U-Net параметризує $\varepsilon_\theta(x_t, t, c)$ або $v_\theta(\cdot)$.

## Порівняння: AF3 diffusion vs image diffusion (U-Net)

| Аспект | AF3 diffusion | Image diffusion (U-Net) |
| --- | --- | --- |
| Простір стану | 3D координати атомів $\mathbb{R}^{N\times3}$ | Пікселі або latent-карти $\mathbb{R}^{H\times W\times C}$ |
| Що денойзимо | Геометрію молекулярного комплексу | Зображення/латент |
| Основна мережа | Геометрично-обумовлений модуль AF3 (з Pairformer-conditioning) | U-Net (encoder-decoder + skip connections) |
| Conditioning | Single/pair репрезентації, типи молекул, шаблони/контекст | Текст, клас, image-conditioning (через cross-attention) |
| Симетрії | Важлива $SE(3)$-узгодженість (оберт/зсув у 3D) | Трансляційна структура 2D-grid; повна обертальна еквіваріантність зазвичай не вбудована |
| Ціль навчання | Шум/score у координатному просторі + структурні обмеження | Шум $\varepsilon$, velocity $v$, або score у 2D/latent просторі |
| Вихід | 3D структура комплексу (білок/РНК/ДНК/ліганд) | Синтезоване зображення |
| Ключові метрики | RMSD, lDDT/pLDDT, DockQ, clash/contact quality | FID, CLIP score, людська оцінка якості |

Коротко: обидва підходи оптимізують зворотний процес дифузії, але **індуктивні упередження (inductive biases)** відрізняються: для AF3 критична 3D-геометрія та фізична узгодженість, для image diffusion — багатомасштабна 2D-структура і семантичний conditioning.

## LoRA для дифузійних backbone-моделей

`LoRA` можна застосовувати і до дифузійних моделей, використовуючи ту саму low-rank ідею:

$$W' = W + \Delta W, \qquad \Delta W = \frac{\alpha}{r}BA$$

### Чому це важливо для diffusion

- Повний fine-tuning `U-Net`, `DiT` або text-conditioning блоків дорогий.
- Багато практичних задач вузькі: стилізація, personalization конкретного об'єкта, доменна адаптація, tuning під окремий control-сигнал.
- Невеликий LoRA-checkpoint простіше зберігати й комбінувати, ніж повну копію diffusion-моделі.

### Типові схеми вставки

| Diffusion backbone | Куди зазвичай ставлять LoRA | Типове застосування |
| --- | --- | --- |
| Latent diffusion `U-Net` | cross-attention `q/k/v/o` проекції, інколи feed-forward або легкі convolutional блоки | стилізація, adaptation під персонажа або об'єкт |
| Diffusion Transformer (`DiT`) | self-attention та MLP-проекції | перенесення домену з transformer-подібним pipeline |
| Multimodal conditioning stack | attention/MLP шари текстового енкодера | зміна prompt understanding або alignment |

### Основні властивості

- LoRA змінює лише вибрані шари, тому базовий denoiser лишається frozen.
- Файли адаптерів малі, їх зручно поширювати окремо від базової моделі.
- Для image diffusion LoRA часто ставлять у шари, що найсильніше впливають на conditioning і prompt-to-image поведінку.
- Для diffusion transformer workflow стає дуже схожим на LoRA в LLM.

### Обмеження

- Надто вузько спеціалізований адаптер може зменшити різноманітність генерації або перенаситити стиль.
- При малому або погано зібраному датасеті часто виникає concept leakage.
- Комбінування багатьох адаптерів може створювати інтерференцію між семантикою prompt і траєкторією denoising.

## Quantization для дифузійних моделей

Квантування дифузійних моделей складніше, ніж для одноразових forward-проходів, бо помилки denoising можуть накопичуватись по timestep-ах.

### Чому diffusion quantization складніше

- Розподіли активацій змінюються залежно від timestep $t$.
- Denoiser викликається багато разів, тому невелика помилка на ранньому кроці може посилитись пізніше.
- Skip/shortcut шляхи можуть мати іншу статистику активацій, ніж основна гілка.

### Основні підходи

| Підхід | Типова ціль | Навіщо застосовується | Головний ризик |
| --- | --- | --- | --- |
| PTQ з timestep-aware calibration | pretrained `U-Net` / `DiT` | не треба дорогого retraining | mismatch калібрування між timestep-ами |
| Mixed precision | чутливі блоки лишаються у вищій точності | краще береже якість зображення | слабше стискає модель |
| Weight-only low-bit inference | важкі ваги denoiser-а | дешевший deployment | активації лишаються дорогими |
| Quantization + fewer-step samplers/distillation | увесь sampling loop | додаткове зменшення latency | складніше втримати якість |

### Трансформери vs diffusion: у чому різниця

| Аспект | Quantization трансформерів | Quantization diffusion-моделей |
| --- | --- | --- |
| Кількість проходів на один sample | зазвичай один decoding step за раз | десятки denoising-кроків |
| Накопичення помилки | локально впливає на наступні токени | накопичується вздовж усієї sampling-траєкторії |
| Об'єкт калібрування | token activations / статистика послідовності | timestep-dependent latent statistics |
| Найчутливіші модулі | attention, MLP, `KV cache` | denoiser-блоки, timestep embeddings, skip paths |

### Практичне правило

- Для transformer-based diffusion backbone (`DiT`) можна повторно використати багато ідей із quantization трансформерів.
- Для `U-Net`-дифузії калібрувати треба на репрезентативних timestep-ах, а не лише на статичних активаціях.
- Починати краще з quantization ваг, а вже потім агресивно стискати активації.
- Якщо якість різко падає, найчутливіші блоки варто повернути у вищу точність.
