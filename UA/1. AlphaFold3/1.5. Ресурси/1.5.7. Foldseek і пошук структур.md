# 1.5.7. Foldseek і пошук структур

[[UA/Головна]] > [[UA/1. AlphaFold3/1.5. Ресурси/1.5.1. Ключові терміни|Ресурси]] > Foldseek і пошук структур
🇬🇧 [[EN/1. AlphaFold3/1.5. Resources/1.5.7. Foldseek and Structure Search|English]]

`Foldseek` — практична система structure search для порівняння великих наборів experimental і predicted structures. В епоху `AF2`/`AF3` це важливо, бо structural prediction є лише половиною workflow; друга половина це пошук подібних folds, assemblies і plausibility analogs у великому масштабі.

## Чому Foldseek важливий

Щойно predicted structure уже є, майже відразу виникають питання:

- чи вже спостерігався схожий fold;
- чи існує related interface в `PDB` або `AlphaFoldDB`;
- чи можна швидко порівняти predicted complex із великою structural collection.

Foldseek важливий саме тому, що робить такий пошук практичним у масштабі баз даних.

## Ключові use cases

| Use case | Чому це важливо |
| --- | --- |
| Fast structure search | Дозволяє знаходити related folds швидше, ніж повільні pairwise structural alignment workflows |
| Multimer search | Дає пошук на рівні assemblies, а не лише окремих chain |
| Clustering | Дозволяє організовувати великі колекції predicted structures |
| Predicted-structure workflows | Корисний разом із `PDB`, `AlphaFoldDB` і локальними prediction outputs |

## Практична цінність для цього vault

Foldseek особливо доречний тут, бо база знань уже змішує:

- experimental structures із `PDB`;
- computed models із `AlphaFoldDB`;
- локальні outputs і PDF-derived references.

Тому note про structure search тут корисніша, ніж суто теоретичне порівняння.

## Обмеження

| Обмеження | Чому це важливо |
| --- | --- |
| Search не дорівнює interpretation | Structural hit все одно потребує biological validation |
| Fast heuristics не є mechanistic proof | Схожі folds не завжди означають shared function |
| Це retrieval, а не generation | Foldseek доповнює prediction models, а не замінює їх |
