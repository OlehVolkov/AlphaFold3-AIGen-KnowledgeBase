## Що таке mmCIF?

**mmCIF** (macromolecular Crystallographic Information File) — офіційний формат Protein Data Bank з 2014 року. Побудований на стандарті **CIF** (Crystallographic Information File), розробленому IUCr. Замінив застарілий PDB формат (`.pdb`), який мав обмеження на кількість атомів та ланцюгів.

### Порівняння mmCIF vs PDB (legacy)

| | mmCIF / `.cif` | PDB legacy / `.pdb` |
|---|---|---|
| Статус | Офіційний стандарт PDB | Застарілий, підтримується |
| Макс. атомів | Необмежено | ~99 999 |
| Макс. ланцюгів | Необмежено | 62 (A-Z, 0-9) |
| Структура | Ключ-значення + таблиці | Фіксована ширина колонок |
| Метадані | Повні (експеримент, автори...) | Обмежені |
| Великі комплекси | ✅ | ❌ |

## Структура mmCIF файлу

```
data_1ABC                          ← блок даних (ім'я структури)

_exptl.method            'X-RAY DIFFRACTION'
_refine.ls_d_res_high    1.74      ← роздільна здатність (Å)
#
loop_                              ← початок таблиці
_atom_site.group_PDB               ← назви колонок...
_atom_site.id
_atom_site.type_symbol
_atom_site.label_atom_id
_atom_site.label_comp_id
_atom_site.label_asym_id
_atom_site.label_seq_id
_atom_site.Cartn_x
_atom_site.Cartn_y
_atom_site.Cartn_z
_atom_site.occupancy
_atom_site.B_iso_or_equiv
ATOM  1  N  N   MET A 1   11.751  26.466  21.000  1.00  30.00  ← рядки даних
ATOM  2  C  CA  MET A 1   12.501  25.322  20.500  1.00  28.00
HETATM 1000 C  C1  HEM A . 15.100  18.200  12.300  1.00  20.00
```

### Ключові поля `_atom_site`

| Поле | Опис | Приклад |
|------|------|---------|
| `group_PDB` | Тип запису | `ATOM` (полімер) / `HETATM` (ліганд) |
| `label_asym_id` | ID ланцюга (авторський) | `A`, `B` |
| `label_comp_id` | Трилітерний код залишку | `MET`, `HEM`, `ATP` |
| `label_seq_id` | Номер залишку | `1`, `42` |
| `label_atom_id` | Назва атома | `CA`, `N`, `O`, `CB` |
| `type_symbol` | Хімічний елемент | `C`, `N`, `O`, `S`, `FE` |
| `Cartn_x/y/z` | Координати (Å) | `11.751` |
| `occupancy` | Заповненість (0–1) | `1.00` |
| `B_iso_or_equiv` | B-фактор / pLDDT у AF3 | `30.00` |

## Читання через BioPython — MMCIFParser

```python
from Bio.PDB import MMCIFParser, MMCIF2Dict
import numpy as np

parser = MMCIFParser(QUIET=True)
structure = parser.get_structure("1abc", "1abc.cif")

# Ієрархія: Structure → Model → Chain → Residue → Atom
model = structure[0]                          # перша модель (NMR може мати кілька)

for chain in model:
    residues  = list(chain.get_residues())
    # Тип залишку: стандартний AA, гетероатом, вода
    aa_res    = [r for r in residues if r.id[0] == " "]
    hetatm    = [r for r in residues if r.id[0].startswith("H")]
    water     = [r for r in residues if r.id[0] == "W"]

    print(f"Ланцюг {chain.id}: "
          f"{len(aa_res)} AA, {len(hetatm)} HETATM, {len(water)} вод")

    for residue in aa_res[:2]:
        print(f"  {residue.resname} {residue.id[1]}")
        for atom in residue:
            # atom.coord → np.array([x, y, z])
            # atom.bfactor → B-фактор
            # atom.occupancy → заповненість
            print(f"    {atom.name:4s}  xyz={atom.coord}  B={atom.bfactor:.1f}")
```

## Читання через gemmi (рекомендовано для великих структур)

`gemmi` у 5-20 разів швидше BioPython і має зручніший API для аналізу.

```python
import gemmi

st = gemmi.read_structure("1abc.cif")

print(f"Назва:        {st.name}")
print(f"Роздільність: {st.resolution:.2f} Å")
print(f"Моделей:      {len(st)}")

model = st[0]
for chain in model:
    print(f"\nЛанцюг {chain.name}: {len(chain)} залишків")
    for res in chain:
        is_ligand = not res.is_water() and res.entity_type != gemmi.EntityType.Polymer
        tag = "[LIG]" if is_ligand else ""
        print(f"  {res.name:4s} {str(res.seqid):5s} {tag}")
        for atom in res:
            print(f"    {atom.name:4s}  "
                  f"({atom.pos.x:7.3f}, {atom.pos.y:7.3f}, {atom.pos.z:7.3f})  "
                  f"B={atom.b_iso:.1f}  occ={atom.occ:.2f}")
```

## Витягання координат як NumPy масив

```python
import gemmi
import numpy as np

def get_backbone_coords(cif_path: str, chain_id: str) -> dict[str, np.ndarray]:
    """
    Повертає координати N, CA, C, O для кожного залишку у вигляді масивів (N, 3).
    """
    st    = gemmi.read_structure(cif_path)
    chain = st[0][chain_id]

    backbone = {"N": [], "CA": [], "C": [], "O": []}

    for residue in chain:
        if residue.entity_type != gemmi.EntityType.Polymer:
            continue
        for atom_name in backbone:
            atom = residue.find_atom(atom_name, "\0")
            if atom:
                backbone[atom_name].append([atom.pos.x, atom.pos.y, atom.pos.z])
            else:
                backbone[atom_name].append([np.nan, np.nan, np.nan])

    return {k: np.array(v) for k, v in backbone.items()}


bb = get_backbone_coords("1abc.cif", "A")
print(f"CA атомів: {(~np.isnan(bb['CA'][:, 0])).sum()}")

# Радіус інерції (міра компактності)
ca = bb["CA"]
ca_clean = ca[~np.isnan(ca[:, 0])]
centroid  = ca_clean.mean(axis=0)
rg        = np.sqrt(((ca_clean - centroid) ** 2).sum(axis=1).mean())
print(f"Радіус гірації Rg = {rg:.2f} Å")
```

## Аналіз контактів на інтерфейсі

```python
import gemmi
import numpy as np
from itertools import product

def find_interface_residues(
    cif_path: str,
    chain_a: str,
    chain_b: str,
    cutoff_ang: float = 5.0,
) -> tuple[list, list]:
    """
    Знаходить залишки на інтерфейсі двох ланцюгів.
    Контакт = будь-яка пара важких атомів ближче ніж cutoff_ang Å.
    """
    st = gemmi.read_structure(cif_path)
    model = st[0]

    def get_atoms(chain_id):
        atoms = []
        for res in model[chain_id]:
            for atom in res:
                if atom.element != gemmi.Element("H"):
                    atoms.append((res, atom))
        return atoms

    atoms_a = get_atoms(chain_a)
    atoms_b = get_atoms(chain_b)

    iface_a, iface_b = set(), set()
    for (res_a, at_a), (res_b, at_b) in product(atoms_a, atoms_b):
        pa = np.array([at_a.pos.x, at_a.pos.y, at_a.pos.z])
        pb = np.array([at_b.pos.x, at_b.pos.y, at_b.pos.z])
        if np.linalg.norm(pa - pb) <= cutoff_ang:
            iface_a.add((res_a.name, str(res_a.seqid)))
            iface_b.add((res_b.name, str(res_b.seqid)))

    return sorted(iface_a), sorted(iface_b)


iface_a, iface_b = find_interface_residues("1abc.cif", "A", "B")
print(f"Інтерфейс A: {len(iface_a)} залишків → {iface_a[:5]}")
print(f"Інтерфейс B: {len(iface_b)} залишків → {iface_b[:5]}")
```

## Парсинг виходу AlphaFold 3

AF3 записує передбачені структури у `.cif` з метриками якості в B-факторі та спеціальних полях.

```python
import gemmi
from Bio.PDB.MMCIF2Dict import MMCIF2Dict

def parse_af3_output(cif_path: str):
    st = gemmi.read_structure(cif_path)
    d  = MMCIF2Dict(cif_path)

    # --- pLDDT по залишках (B-фактор = pLDDT у виходах AF3) ---
    print(f"\n{'Ланцюг':<8} {'Залишок':<10} {'pLDDT':>6}")
    print("-" * 28)
    for chain in st[0]:
        for res in chain:
            ca = res.find_atom("CA", "\0")
            if ca:
                category = (
                    "🔵 висока"   if ca.b_iso >= 70 else
                    "🟡 середня"  if ca.b_iso >= 50 else
                    "🟠 низька"
                )
                print(f"{chain.name:<8} {res.name} {str(res.seqid):<6} "
                      f"{ca.b_iso:>6.1f}  {category}")

    # --- Середній pLDDT по ланцюгах ---
    print("\n--- Середній pLDDT ---")
    for chain in st[0]:
        scores = [res.find_atom("CA", "\0").b_iso
                  for res in chain if res.find_atom("CA", "\0")]
        if scores:
            print(f"  Ланцюг {chain.name}: {sum(scores)/len(scores):.1f}")
```

## Завантаження структури з PDB

```python
import gemmi
import urllib.request
from pathlib import Path

def fetch_pdb(pdb_id: str, save_dir: str = ".") -> str:
    """Завантажує mmCIF файл з RCSB PDB."""
    pdb_id = pdb_id.lower()
    url  = f"https://files.rcsb.org/download/{pdb_id}.cif"
    path = Path(save_dir) / f"{pdb_id}.cif"

    if not path.exists():
        urllib.request.urlretrieve(url, path)
        print(f"Завантажено: {path}")
    else:
        print(f"Вже є: {path}")
    return str(path)


cif_path = fetch_pdb("1ABC")
st = gemmi.read_structure(cif_path)
print(f"Завантажено: {st.name}, роздільність {st.resolution:.2f} Å")
```

## Запис / модифікація mmCIF через gemmi

```python
import gemmi

st = gemmi.read_structure("input.cif")

# Видалити молекули води
for model in st:
    for chain in model:
        chain.remove_waters()

# Зберегти лише ланцюг A
model = st[0]
chains_to_remove = [c.name for c in model if c.name != "A"]
for name in chains_to_remove:
    model.remove_chain(name)

# Записати
st.update_mmcif_block()
st.make_mmcif_document().write_file("chain_A_only.cif")
print("Збережено chain_A_only.cif")
```

## Порівняння FASTA та mmCIF

| | FASTA | mmCIF |
|---|---|---|
| Вміст | Послідовність (1D) | Повна 3D структура |
| Метадані | Мінімальні | Повні (метод, роздільність, автори) |
| Розмір | Кілобайти | Мегабайти |
| Використання в AF3 | **Вхід** | **Вихід** |
| Основні бібліотеки | `BioPython SeqIO` | `gemmi`, `BioPython PDB` |
| Швидкість парсингу | Миттєво | `gemmi` >> BioPython |
