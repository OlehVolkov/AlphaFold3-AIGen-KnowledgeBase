# 3.7. Boltz-1

[[UA/Головна]] > [[UA/3. Моделі/3.0. Огляд моделей|Моделі]] > Boltz-1
🇬🇧 [[EN/3. Models/3.7. Boltz-1|English]]

`Boltz-1` — відкрита модель для biomolecular interaction modeling, яку зручно розглядати як `AF3`-adjacent систему для передбачення 3D біомолекулярних комплексів. Її головна цінність у тому, що вона зміщує multimolecular complex prediction від server-first парадигми до відкритого локального stack.

## Чому Boltz-1 важливий

Багато дослідників хочуть можливості рівня `AF3`, але також потребують:

- відкритих ваг;
- trainable коду;
- локального inference;
- inspectable data і benchmark pipeline.

Boltz-1 важливий саме тому, що намагається дати все це в одному open package.

## Практична роль у локальному stack

Boltz-1 найкраще розуміти як open generalist для biomolecular interactions:

- ширший за чисто protein-only модель;
- локальніший і модифікованіший, ніж server-only workflow;
- природна точка порівняння поруч із `AF3` і `Chai-1`.

## Сильні сторони

| Сильна сторона | Чому це важливо |
|---|---|
| Відкриті код і ваги | Легше inspect, benchmark і automate локально |
| Фокус на biomolecular complexes | Корисно для білків, нуклеїнових кислот і лігандів в одному workflow |
| MIT license | Висока практична цінність для reuse в research software |
| Fast-moving ecosystem | Добре підходить для open benchmarking і engineering-ітерацій |

## Коли Boltz-1 корисний

- коли потрібен open `AF3`-adjacent baseline;
- коли local inference та automation важливіші за server workflow;
- коли команді потрібен modifiable stack для biomolecular complexes;
- коли пряме порівняння з `Chai-1` або `AF3` входить у план оцінки.

## Обмеження

| Обмеження | Чому це важливо |
|---|---|
| Література на стадії preprint | Claims слід трактувати обережніше, ніж у давно усталених peer-reviewed baseline |
| Швидко змінюване ПЗ | Практична поведінка може змінюватися швидше, ніж у класичних benchmark papers |
| Більша складність, ніж у protein-only stack | Multimolecular flexibility підвищує operational complexity |
