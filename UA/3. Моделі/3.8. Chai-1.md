# 3.8. Chai-1

[[UA/Головна]] > [[UA/3. Моделі/3.0. Огляд моделей|Моделі]] > Chai-1
🇬🇧 [[EN/3. Models/3.8. Chai-1|English]]

`Chai-1` — multi-modal foundation model для molecular structure prediction. Практично це один із найважливіших open local alternatives для передбачення комплексів, що змішують білки, малі молекули, DNA, RNA, glycosylations і user-provided restraints.

## Чому Chai-1 важливий

`AF3` зробив broad biomolecular interaction modeling масовим, але локальним workflow часто все ще потрібні:

- відкриті код і ваги;
- programmable inference;
- явний контроль над templates, MSAs і restraints;
- шлях до локального deployment на GPU.

Chai-1 важливий саме тому, що пакує ці потреби в практичний open stack.

## Основні можливості

| Можливість | Практичний зміст |
| --- | --- |
| Multi-modal input | Білки, ліганди, DNA, RNA, glycosylations та інше |
| Local inference | Зручно для scripted batch workflows і reproducible pipelines |
| Підтримка MSA | Може працювати з default embeddings, але також підтримує richer MSA input |
| Restraints і custom contexts | Корисно для guided modeling та integrative setups |

## Коли Chai-1 корисний

- коли потрібен локальний stack для complex prediction замість hosted service;
- коли моделювання вимагає додаткових restraints або custom template control;
- коли один workflow повинен працювати з білками разом із лігандами та нуклеїновими кислотами;
- коли потрібен open benchmark проти `AF3` або `Boltz-1`.

## Сильні сторони

- **Широке entity coverage** в одному workflow;
- **Практичний local interface** через CLI і Python entry points;
- **Відкриті ваги та код** під permissive license;
- **Кращий fit для automation**, ніж server-only workflow.

## Обмеження

| Обмеження | Чому це важливо |
| --- | --- |
| Вимогливість до hardware | Великі комплекси все ще потребують потужних GPU |
| Швидко змінювана реалізація | Практичні деталі можуть швидко змінюватися в активному repo |
| Молодий benchmark target | Реальну поведінку треба перевіряти окремо для кожного task class |
